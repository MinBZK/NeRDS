# 6. Make safe systems

Security is a fundamental aspect of digital systems. It is not just about protecting against attacks, but also about ensuring the availability, integrity and confidentiality of data and services.

## Why is it important?

- **Protection of sensitive data**: Public authorities often process personal and sensitive information
- **Conservation of trust**: Citizens should be able to trust that their data is safe
- **Legal obligation**: Compliance with relevant legislation and standards
- **Continuity of services**: Avoid disruption of essential public services
- **Protection against threats**: Defense against a growing number of cyber threats
- **Cost control**: Avoid costly recovery costs and fines following security incidents
- **Reputation protection**: Maintain public and partner trust
- **Competitive advantage**: Strong security enables innovation and cooperation

By taking security seriously and integrating it into all aspects of technology development and use, you not only protect data but also citizens' confidence in the digital government.

1. **Consider security from the beginning**:

- Assess security risks before starting a technology program
- Identify potential vulnerabilities in systems and data
- Determine which security expertise and resources are needed

2. **Implement proportional security measures**:

- Protect information technology and digital services
- Ensure appropriate access to user data
- Evaluate and update security protocols constantly

3. **Focus on multiple security dimensions**:

- Network and infrastructure security
- Data security
- Service security
- Cloud security

### Critical security strategies

- **Identify, protect, detect, respond and recover**: Follow this security incident cycle
- **Use security checks like**:
- Data encryption
- Single sign-on
- Two factor authentication
- Specific access control
- Use monitoring
- Timely system patching

Implementation of security measures

- **Riscoanalysis**: Identify threats and vulnerabilities
- **Authentication and authorization**: Implement strong access controls
- **Encryption**: Protect data during storage and transport
- **Safe development practices**: Follow secure coding standards
- **Penetration tests**: Regularly test system security
- **Incident response**: Develop and test contingency response plans
- **Delivery management**: Evaluate and check vendor security

### Continuous Security Management

- Set up plans for continuous improvement
- Review security checks regularly
- Monitor potential risks
- Adapt to changing technological landscapes

### Recommended Actions

- Perform comprehensive risk assessments
- Assign clear security roles and responsibilities
- Document security processes
- Create user access training and control plans
- Integrate security considerations into general programme planning
